\begin{onlyenv}<2|handout:2>
\vspace{-0.9cm}

\begin{block}{Preprocessing}
Le préprocesseur effectue différentes opérations de substitution
et de supression dans le code :
\begin{itemize}
\item Supression des commentaires (\bvrb|//| et  \bvrb|/* */|)
qui sont utiles au programmeur, mais inutiles pour le processeur.\\
\item Inclusion des fichier \Verb|.h| dans le fichier \Verb|.c| 
(directive \bvrb|#include|). Ici, il permet de donner le prototype
de la fonction \bvrb|printf| (son format).\\
\item Traitement des directives de compilation qui commencent par
un caractère \bvrb|#| (voir plus loin).\\
\end{itemize}

\end{block}
\end{onlyenv}

\begin{frame}[fragile]
\frametitle{La directive \bvrb|\#define|}
%\frametitle{L'instruction \bvrb|if...else|}


\begin{itemize}
\item Déclaration de constantes (ou d'une expression fixe quelconque) :\\
\bvrb|#define £textitµidentificateur reste_de_la_ligne§|\\
\item Lorsque le préprocesseur lit une ligne de ce type, il remplace
toutes les occurences suivantes de \bvrb|£textitµidentificateur§| dans le fichier texte
par \bvrb|£textitµreste_de_la_ligne§|\\
\item Par convention, on écrit l'identificateur en MAJUSCULE.
\end{itemize}

\begin{frame}[fragile]
\frametitle{Conseils sur le \bvrb|\#define|}
\begin{itemize}
\item Il est fortement recommandé de placer les
\bvrb|#define| au début de votre programme au même endroit
que les \bvrb|#include|.\\
\item Limitez l'utilisation des \bvrb|#define| à des options de
compilation ou à des constantes valables et utilisées dans tout le
programme.\\
\end{itemize}

\begin{frame}[fragile]
\frametitle{Les directives pour le préprocesseur}
\begin{itemize}
\item \bvrb|#define £textitµidentificateur reste_de_la_ligne§|\\
Remplace \bvrb|£textitµidentificateur§| par  \bvrb|£textitµreste_de_la_ligne§|
jusqu'à la fin du programme ou jusqu'à une instruction \bvrb|#undef|.\\
\vspace{0.5cm}
\item \bvrb|#undef £textitµidentificateur§|\\
Marque la fin du remplacement systématique initié par \bvrb|#define|\\
\vspace{0.5cm}

\item \bvrb|#ifdef £textitµidentificateur§ ... #endif|\\
Inlus la partie de programme située entre \bvrb|#ifdef| et \bvrb|#endif| si
l'identificateur a été déclaré avant dans un \bvrb|#define|. Sinon, la partie de
programme concernée est supprimée avant compilation.\\
\end{itemize}

\end{frame}
