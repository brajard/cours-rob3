\begin{frame}[fragile]
\frametitle{Les entiers naturels (positifs)}
Ils correspondent aux types \bvrb|unsigned int|, 
\bvrb|unsigned short|, ...

\begin{block}{}
Il sont codés en binaire selon un nombre fixé de bits.
\end{block}

Exemple su 16 bits (taille minimum du type \bvrb|short|) :\\
Le codage de $\mathbf{13}$ est $\mathbf{0000000000001101}$. 

\begin{alertblock}{Limites}
Si le codage utilise \textbf{n} bits, on peut donc coder tous le entiers de
\textbf{0} à $\mathbf{2^n-1}$.\\
Sur 16 bits, on code tous les entiers de \textbf{0} à \textbf{65 535}.
\end{alertblock}

\end{frame}

\begin{frame}[fragile]
\frametitle{Les entiers relatifs}
Ils correspondnent aux types \bvrb|short|, \bvrb|int|, ...
\begin{block}{}
\begin{itemize}
\item Le bit de gauche (ou "bit de poids fort") représente le signe
\begin{itemize}
\item 0 pour les entiers positifs
\item 1 pour les entiers négatifs
\end{itemize}
\item Les \textbf{n-1} autres bits code l'entier
\begin{itemize}
\item Les entiers positifs sont codés en binaire comme les entiers non signés
\item Les entiers négatifs sont codés en complément à deux.
\end{itemize}

\end{itemize}

\end{block}
\end{frame}

\begin{frame}
\frametitle{Les entiers relatifs positifs}
\begin{block}{}
Ils sont codés exactement comme les entiers non signés (mais avec le bit de gauche à 0)\\
\end{block}

Le codage de 13 est

\begin{frame}
\frametitle{Les entiers relatifs négatifs}
\begin{block}{}
Ils sont codés en complément à deux par rapport à l'entier positif correspondant.
\end{block}
\begin{columns}
\column{0.5\textwidth}
Pour coder un entier relatif :
\begin{enumerate}
\item<2-> On code d'abord l'oposé (positif) du nombre
\item<4-> On complémente chaque bit (négation
ou complément à 1)
\item<5-> On ajoute 1
\end{enumerate}

Ex : $-13 + 13 = 0$\\
\vspace{1em}
\begin{tabular}{lc}
&1111111111110011 \\
+ & 0000000000001101 \\
\cline{2-2}
& 0000000000000000\\
\end{tabular}

\end{alertblock}

\begin{frame}
\frametitle{Représentation des réelles}
\framesubtitle{L'écriture binaire des décimales}
La décomposition d'un nombre décimale se fait similairement aux entiers avec des puissances de 2\\
Exemple :
$$
5.25 = 
 \tikz[baseline,remember picture]{
   \node[fill=red!20,ellipse,anchor=base] {$1$};
 }
\times 2^2 + 
 \tikz[baseline,remember picture]{
   \node[fill=red!20,ellipse,anchor=base] {$0$};
 }
\times 2^1 + 
 \tikz[baseline,remember picture]{
   \node[fill=red!20,ellipse,anchor=base] {$1$};
 }
\times 2^0 +
  \tikz[baseline,remember picture]{
   \node[fill=green!20,ellipse,anchor=base] {$0$};
 }
\times 2^{-1} + 
  \tikz[baseline,remember picture]{
   \node[fill=green!20,ellipse,anchor=base] {$1$};
 }
\times 2^{-2}
$$

L'écriture binaire de 5.25 est donc 101.01

\begin{block}{L'écriture normalisée}
Soit z un nombre décimal, son écriture normalisée est la suivante :\\
\vspace{0.5em}
$
z = \pm   1,XXXXXXX \times 2^n
$
\begin{itemize}
\item $\pm$ est le signe
\item $XXXXXXX$ est la mantisse
\item $n$ est l'exposant
\end{itemize}
\end{block}
La forme normalisée de 5.25 est donc $1.0101 \times 2^2$
\end{frame}

 \begin{frame}
 \frametitle{Représentation des réelles}
 \framesubtitle{La norme IEEE 754 (sur 32 bits, simple précision)}

 \begin{block}{Expression des décimaux}
$z = -1^\mathbf{S} \times 2^{\mathbf{E} - 127} \times (1+\mathbf{M})$\\
 \begin{itemize}
\item $\mathbf{S}$ code le signe sur \red{1 bit}
\item $\mathbf{E}$ code l'exposant sur \red{8 bits}
\item $\mathbf{M}$ code la mantisse sur \red{23 bits}
\end{itemize}
 \end{block}

Exemple : $5.25$\\
Ecriture normalisée : $1.0101\times2^2$
\begin{enumerate}
\item Le bit  $\mathbf{S}$ vaut \textbf{0} car le nombre est positif.
\item $\mathbf{E}$ vaut 129, car l'exposant $\mathbf{E} - 127 = 2$ (en binaire E s'écrit \textbf{01000001})
\item La mantisse $\mathbf{M}$ vaut \textbf{01010000000000000000000}
\item Le nombre 5.25 se code sur 32 bits par :\\
\textbf{0 01000001 01010000000000000000000}
\end{enumerate}
 \end{frame}

\begin{frame}
\frametitle{Pourquoi est-ce important ?}
\begin{columns}
\column{0.65\textwidth}
Cela peut entraîner des problèmes lorsque l'on dépasse la capacité du codage.
(voir Ariane 5)
\begin{exampleblock}{}
{\footnotesize
Pour illustrer ce problème, dans cet exemple, la mantisse est codée sur \red{5} bits et l'exposant
$\mathbf{E}-7$ est codé sur \red{4 bits}.
}
\begin{itemize}
\item Le nombre \textbf{14} se code \textbf{0 1010 11000}
\item Le nombre \textbf{3.25} se code \textbf{0 1000 10100}
\item La somme des deux (\textbf{17.25}) dépasse les capacités, car la mantisse nécessiterait 6 bits(000101).
Le résultat calculé par la machine serait \textbf{17}.
\end{itemize}
Note : Par contre il est possible d'écrire \textbf{18}(\textbf{0101100100}) avec ce codage.
\end{exampleblock}
