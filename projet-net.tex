\begin{frame}
\frametitle{Le but du jeu}
Vous êtes deux pirates (hacker) dans un réseau informatique. \\
\vspace{2em}
\begin{alertblock}{}
L'objectif est d'être le premier à pirater (capturer) le c\oe ur du
réseau.
\end{alertblock}
\end{frame}

\begin{frame}
\frametitle{Comment faire ?}
\begin{figure}
\includegraphics[width=.7\textwidth]{./fig/board.png}
\end{figure}
Vous êtes le joueur bleu ou le joueur rouge, et vous devez capturer le
noeud central (c\oe ur du réseau en vert).
\end{frame}

\begin{frame}
\frametitle{Que trouve-t-on dans le réseau ?}
\begin{columns}
\column{.55\textwidth}
Il y a 3 types d'emplacement dans le réseau :
\begin{itemize}
\item Les n\oe ud de réseau qui peuvent être de type 1 (n\oe ud \nd), type 2
  (n\oe ud \ns) ou de type 3 (n\oe ud \nc) ;
\item Des liens entre deux n\oe uds voisins qui peuvent être horizontales ou
  verticales ;
\item Emplacements vides.
\end{itemize}
\column{.42\textwidth}
\begin{figure}
\includegraphics[width=\textwidth,trim={0 0 7.5cm 1.1cm},clip]{./fig/board.png}
\end{figure}
\end{columns}
\end{frame}

\begin{frame}
\frametitle{Que connaissez-vous au début du jeu ?}
\begin{columns}
\column{.35\textwidth}
\begin{itemize}
\item L'intégralité du réseau (position des n\oe uds et des liens)
\item La position des n\oe uds  déjà capturé par les joueurs au début
  du jeu et la position du c\oe ur du réseau.
\item Le joueur qui commence.
\item Votre niveau initial d'énergie.
\end{itemize}
\column{.65\textwidth}
\begin{figure}
\includegraphics[width=\textwidth]{./fig/board.png}
\end{figure}
\end{columns}
\end{frame}

\begin{frame}
\frametitle{Deux types d'action}
\framesubtitle{1- Capture de n\oe ud}

\begin{block}{}
A chaque tour,  vous pouvez essayer de capturer un n\oe ud du réseau
si :
\begin{itemize}
\item il y a un lien entre ce n\oe ud et un n\oe ud que vous avez déjà capturé.
\item Le n\oe ud que vous tentez de capturer ne doit pas être déjà
  capturé ou en cours de capture.
\end{itemize}
\end{block}

Le temps de capture d'un n\oe ud dépend du type de n\oe ud :
\begin{itemize}
\item 1 tour pour les n\oe uds \nd ~de type 1 ;
\item 2 tours pour les n\oe uds \ns ~de type 2 ;
\item 3 tours pour les n\oe uds \nc ~de type 3.
\end{itemize}

\begin{alertblock}{}
A chaque fois, vous gagnez 
une unité d'énergie
\end{alertblock}

\end{frame}

\begin{frame}
\frametitle{Deux types d'action}
\framesubtitle{2- Création/Destruction de liens}
\begin{block}{}
A chaque tour vous pouvez dépenser 5 unités d'energie créer ou
détruire un lien.
\end{block}
\begin{alertblock}{}
Il faut que l'emplacement choisi soit vide si vous souhaitez
  créer un lien et il faut qu'il soit occupé par un lien si vous
  souhaitez supprimer un lien.
\end{alertblock}
\end{frame}

\begin{frame}
\frametitle{Conditions de victoires/défaites}
\begin{exampleblock}{Vous gagnez si :}
\begin{itemize}
\item Vous capturer le n\oe ud central en premier.
\item Votre adversaire perd avant.
\end{itemize}
\end{exampleblock}
\begin{alertblock}{Vous perdez si : }
\begin{itemize}
\item Votre adversaire capture le n\oe ud central avant vous;
\item Vous faites un coup illégal (ex : capture d'un n\oe ud non voisin
  d'un n\oe ud déjà capturé);
\item Vous tentez de créer/supprimer un lien mais vous n'avez pas
  assez d'énergie.
\item Vous ne respectez pas le séquencement du programme (cf document détaillé);
\item Vous mettez trop de temps à jouer.
\end{itemize}

\end{alertblock}
\end{frame}


\begin{frame}
\frametitle{Qui joue ?}

\begin{block}{}
C'est le programme informatique que vous avez programmé qui jouera au jeu et
une fois le programme lancé il n'y a pas d'intervention "humaine".
\end{block}
En pratique, deux programmes jouent l'un contre l'autre en se connectant chacun à 
un serveur qui vérifiera la légalité des coups joués. La connection au serveur se fera 
par l'appel de fonctions d'une bibliothèque qui vous sera fournie.
\begin{alertblock}{}
 Le processus de programmation deva se faire en plusieurs étapes successives:
 \begin{enumerate}
 \item Programme qui ne fait rien ("do nothing")
 \item Programme où vous indiquez "à la main" le coup que vous voulez jouer.
 \item Programme qui joue de façon aléatoire
 \item Programme qui jour de façon intelligente
 \item (si possible) Programme qui jour de façon très intelligente.
 \end{enumerate}
\end{alertblock}

\end{frame}

\begin{frame}
\frametitle{Et maintenant ?}
\begin{block}{}
Vous devez constituer des binômes pour proposer le meilleur programme qui sera présenté 
devant votre groupe la semaine du 8 janvier (pour les ROB et les MAIN). Plus tard, un tournoi sera
organisé entre tous les programmes pour déterminer \alert{LE} programme gagnant.
\end{block}
Evaluation :
\begin{itemize}
\item Evaluation continue sur la réalisation des étapes décrites précédemment. 
\item Evaluation sur la soutenance orale de début Janvier
\item Participation au tournoi.
\end{itemize}

\end{frame}
