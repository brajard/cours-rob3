\begin{frame}[fragile]
\frametitle{Qu'est-ce qu'un tableau}
\onslide<1->{\begin{block}{Point de vue algorithmique}
Un tableau est un ensemble de données de même type.
\end{block}}
\onslide<2->{\begin{block}{Point de vue langage C}
Un tableau est un \red{pointeur} sur le premier élément d'un
ensemble de données de même type.
\end{block}}

\pause[3]
\begin{columns}
\column{0.64\textwidth}
\centering

\begin{frame}[fragile]
\frametitle{Allocation dynamique de mémoire pour un tableau}
\begin{block}{Allocation}
On peut affecter dynamiquement la mémoire d'un tableau :
\begin{itemize}
\item fonction \bvrb|malloc| de la bibliothèqe \bvrb|stdlib.h| :\\
\bvrb|£textitµnom_pointeur§ = (£textitµtype§*)malloc(£textitµespace_memoire§);|
\item \bvrb|espace_memoire| = \red{taille du tableau $\times$ taille du type}
\end{itemize}
\end{block}
Exemple pour un tableau de 10 entiers :
%\centering
\begin{codeblock}{}
\vspace{-.3cm}
\lstset{escapeinside={§§}}
%\lstset{basicstyle=\scriptsize}
\begin{codeC}
int *Tab ;
Tab = (int *) malloc (10*sizeof(int));
\end{codeC}
\vspace{-.3cm}
\end{codeblock}

\begin{block}{Libération}
Lorsque la mémoire n'est plus utilisée, il faut penser à la libérer :
\bvrb|free(£textitµnom_pointeur§);|

\end{block}



\end{frame}

\begin{frame}[fragile]
\frametitle{Exemple}
\begin{codeblock}{}
\vspace{-.3cm}
\lstset{escapeinside={§§}}
%\lstset{basicstyle=\scriptsize}
\begin{codeC}
#include <stdio.h>
#include <stlib.h>

void init (int *Tab, int n);

int main() 
{
   int *Tab;
   int n;
   printf("Entrez la taille du tableau : ");
   scanf("%d",&n);
   Tab=(int *)malloc(n*sizeof(int));
   init(Tab, n);
   ...
   free(Tab);
   return (0);
}
\end{codeC}
\vspace{-.3cm}
\end{codeblock}
\end{frame}

\begin{frame}[fragile]
\frametitle{Problèmes d'allocation mémoire}
Le système ne peut plus allouer de mémoire ?
\begin{itemize}
\item Des zones réservées n'ont pas été libérées ;\\
\item La taille des zones demandées est trop grande.\\
\end{itemize}

\vspace{2em}
Il faut vérifier que la zone mémoire est bien valide 
(test sur la nullité du pointeur sur cette zone).

\begin{codeblock}{}
\vspace{-.3cm}
\lstset{escapeinside={§§}}
\lstset{basicstyle=\scriptsize}
\begin{codeC}
Tab=(int *) malloc (n*sizeof(int));
if (Tab == NULL) {
//Traitement de l'erreur
printf("Erreur dans l'allocation mémoire");
}
\end{codeC}
\vspace{-.3cm}
\end{codeblock}

\end{frame}

\begin{frame}
\frametitle{Allocation statique Vs Allocation dynamique}
\begin{block}{Statique}
\begin{itemize}
\item La mémoire est spécifiée dans le code ;
\item Elle est réservée lors de la compilation ;
\item Il n'y a pas d'allocation à l'exécution 
(pas de problème d'allocation lors de l'exécution)
\end{itemize}
\end{block}

\begin{block}{Dynamique}
\begin{itemize}
\item Le progammeur gère sa mémoire ;
\item \red{Il n'y pas besoin de connaître à l'avance (à la compilation)
la taille du tableau.}
\end{itemize}
\end{block}
\end{frame}

\begin{frame}[fragile]
\frametitle{Les tableaux en 2 dimensions}
\begin{block}{}
Un tableau en deux dimensions peut être vu comme un tableau de tableau
et donc un pointeur sur un pointeur...
\end{block}
\begin{codeblock}{}
\vspace{-.3cm}
\lstset{escapeinside={§§}}
\lstset{basicstyle=\scriptsize}
\begin{codeC}
int **Tab2D ;
\end{codeC}
\vspace{-.3cm}
\end{codeblock}
\vspace{1em}
Il faut donc alluer la mémoire pour le tableau principal et pour
chacun des "sous-tableaux".
\begin{codeblock}{}
\vspace{-.3cm}
\lstset{escapeinside={§§}}
\lstset{basicstyle=\scriptsize}
\begin{codeC}
int nblignes=3,nbcolonnes=4;

Tab2D=(int **)malloc(nblignes*sizeof(int *));

for (i=0;i<nblignes;i++) {
  Tab2D[i]=(int *)malloc(nbcolonnes*sizeof(int));
}
\end{codeC}
\vspace{-.3cm}
\end{codeblock}
\end{frame}
