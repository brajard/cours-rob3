\begin{frame}
\frametitle{Problème posé}
\begin{block}{Problème du Labyrinthe}
Sur une grille, on cherche à trouver le chemin le plus court du point D au point A
sachant qu'il y a des obstacles.
\end{block}
\begin{columns}
\column{0.55\textwidth}
Ce qu'on connaît :
\begin{itemize}
\item Les noeuds accessibles (en blanc) et les noeuds inaccessibles (en noir)
\item Les voisins de chaque noeud (\textit{dans notre ex : noeuds du haut, du bas, de gauche, de droite}).
\item Le coût pour aller d'un noeud à son voisin (\textit{dans notre exemple  : 1 pour tous les noeuds}).
\end{itemize}

\begin{frame}
\frametitle{Un algorithme possible : A*}
\begin{block}{}
L'algorithme A* permet de trouver rapidement le chemin le plus court.
Il s'agit de l'extension d'un autre algorithme : l'algorithme de Dijkstra.
\end{block}

Il est très utilisé en intelligence artificielle (déplacement de robot, jeux vidéos).

\begin{alertblock}{}
Il est nécessaire d'introduire des données supplémentaires au problème initial.
\end{alertblock}

\end{frame}

\begin{frame}[fragile]
\frametitle{Quelles informations supplémentaires ajouter ?}
\begin{block}{La structure case}
Plusieurs champs doivent être indiquées dans une case :
\begin{itemize}
\item Les coordonnées : $x,y$
\item Le coût pour se rendre du départ à cette case : cost
\item Une estimation du coût total (=cost + estimation de la distance restante) : heuristique
\item Les coordonnées du noeud d'où l'on vient.
\end{itemize}
\end{block}
\begin{codeblock}{}
\vspace{-.3cm}
\lstset{escapeinside={§§}}
%\lstset{basicstyle=\scriptsize}
\begin{codeC}
struct case {
  int x,y; 
  int cost ; 
  int heuristique ;
  int xp,yp ; //noeud précédent
  } ;
  
\end{codeC}
\vspace{-.3cm}
\end{codeblock}
\end{frame}

\begin{frame}
\frametitle{Calcul de l'heuristique}
\begin{columns}
\column{.48\textwidth}
\begin{block}{Estimation de la distance restante}
Généralement, elle est estimée comme le coût du chemin le plus court sans obstacle
("à vol d'oiseau").
\end{block}
Légende :
\begin{itemize}
\item D désigne le point de départ
\item A désigne le point d'arrivée
\item $\leftarrow$ désigne la case précédente
\item le couple de nombres définit les valeurs (cout,heuristique).
\end{itemize}

\begin{frame}
\frametitle{Les différentes listes}
Dans l'algorithme A*, on sépare les cases en différentes catégories :
\begin{itemize}
\item Les cases non explorées : ces cases sont les cases pour lesquels on n'a pas encore calculé
la valeur (cout, heuristique, etc.)
\item Les cases à explorer : ces cases sont stockées dans une liste, l'\textcolor{green}{openList}. On a calculé leurs valeurs mais on ne les a pas encore explorées.
\item Les cases déjà explorées : ces cases sont stockées dans une liste, la \textcolor{red}{closedList}. On a calculé leurs valeurs et aussi les valeurs de leurs voisines.
\end{itemize}

\end{frame}


\begin{frame}
\frametitle{Algorithme A*}
\begin{columns}
\column{.48\textwidth}
\begin{itemize}
\item \alert<2>{\textcolor{red}{closedList} = \{\}, \textcolor{green}{openList} = D}
\item Tant que openList non vide
\begin{itemize}
\item \alert <3-18>{c : prendre le plus petit élément de openList et l'ajouté à la closedList.}
\item \alert <18> {si c est l'objectif A : reconstituer le chemin et le renvoyer}
\item \alert <3-17>{sinon : ajouter les voisins de c non explorés dans l'OpenList}
\end{itemize}
\item Si l'openList est vide: pas de chemin possible.
\end{itemize}

\begin{frame}[fragile]
\frametitle{Comment ajouter un voisin dans l'openList ?}
\begin{codeblock}{}
\vspace{-.3cm}
\lstset{escapeinside={§§}}
\lstset{basicstyle=\small}
\begin{codeC}
struct case nouvelle_case (struct case c,int x, int y) {

  struct case v; //case voisine de c
  v.x = x;
  v.y = y;
  v.cost = c.cost+1;
  v.heuristique = v.cost + estimation_distance(x,y);
  v.xp = c.x;
  v.yp = c.y;
  return v;
}
\end{codeC}
\vspace{-.3cm}
\end{codeblock}
